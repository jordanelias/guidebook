The shape is front-loaded, then it collapses. Sourcing (collection) is essentially finished — 784/823 sources verified, metadata near-complete — but almost nothing has been adjudicated into determinations (judgment, 4%) and only 1 parameter has a real synthesis reasoning doc (synthesis, 1%). Render faithfully mirrors that emptiness: it can only present the 14 determinations that exist. This is scaffolding with the shelves stocked but the book largely unwritten — consistent with the project's pre-launch, mid-rehabilitation state.

Read the flags with care: the item-facing corpus is retracted, not finished

Every one of the 87 items that points at a BPC slug points at one whose evidence_state = RETRACTED-PRE-REHAB. Those slugs still carry a stale bpc_complete = 1 flag from the pre-rehabilitation era — so a naïve flag count would report the BPC layer as largely "complete," when the honest current state is that 0 item-facing syntheses are live. The 10 PARTIAL slugs genuinely in progress are cross-cutting methodology/infrastructure units, none linked to a design parameter. The gauges above use current-state signals (reasoning docs, materialized cells), not the stale completion flags.

How these numbers were derived
Read-only queries against data/guidebook.db (user_version 32, 249 data migrations). Denominators: research/synthesis over 83 bpc_metadata slugs; collection over 823 evidence_sources; judgment/render over 363 applicable item×population pairs (item_population_links). Determinations from evidence_cell_state; synthesis from reasoning_doc_citations; enforcement coverage from governance/pipeline-contract.yaml.
Two things to keep in mind
(1) "Completeness" here means throughput, not correctness — a stated cell has cleared the gate, not been peer-reviewed. (2) bpc_complete and similar flags are pre-rehabilitation artifacts and overstate reality; current-state signals are used throughout.
