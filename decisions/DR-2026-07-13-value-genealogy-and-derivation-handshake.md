## Context — the two problems are one problem

**Problem 1 (genealogy).** The project's central adjudication — best practice (dignity-bearing, most accommodating) vs consensus/precedent — currently has no computable form. The corridor-width genealogy proves the stakes: 1500mm turning "circles" in 24 jurisdictions trace to one 1970s anthropometric root (REF-00059/60: "based on research conducted in the 1970s… need to be updated"); the 1800mm passing consensus has **no traceable root anywhere in the corpus**; while 2440/2400mm has **three independent roots** (ASL proxemics Co-1, swept-path anthropometry T3 entire-sample, MEXT children's power chairs T5). Precedent counts documents; evidence counts roots — and today nothing in the data layer can count roots. Worse, values are being compared across incommensurable **measurement paradigms** (static turning circle vs dynamic swept-path envelope — 97% of turns are asymmetric spin turns; scooters have no pivot at all) and across **device classes** (multiple codes self-declare their figures manual-wheelchair-only).

**Problem 2 (handshake).** The corpus runs two derivation paths that never reconcile: **top-down** (population/community — BPCs, Co-1, every DB join) and **bottom-up** (function — the 38-file ICF-indexed `references/fdr/` corpus, deliberately population-blind at collection; the functional-deficit-auditor's population↔ICF mapping, which lives as unvalidated skill prose). No mechanism requires the paths to agree before a value ships; dual derivation is undetectable by query. The two corridor requirements (device envelope, d465; signing envelope, d310+DEAF) sit in the same directory with no common key — and DEAF is absent from E-08's `item_population_links` despite holding the anchor evidence, which is precisely the population-silo failure the armature's reverse-mapping (Q4 — resolved on paper 2026-04-28, never built) exists to prevent.

They are one problem because both reduce to: **a claim's authority derives from its derivation, and derivations must be recorded, independent-counted, and cross-checked.** Convergence across independent *evidence axes* is already doctrine (#3); this DR extends the identical logic down to roots (genealogy) and up to derivation paths (handshake).

## Proposed decision — six separately ratifiable items

### H1 — Genealogy fields on `source_value_extractions`

Extend the (empty) B6 substrate with: `root_id` (shared identifier for one evidentiary root), `root_type` CHECK (`measurement_primary` / `participatory_finding` / `committee_assertion` / `derived_calculation` / `untraced`), `root_ref_id` (FK to `evidence_sources` when the root document is in corpus), `echo_of` (transmission-chain note), `measurement_paradigm` CHECK (`swept_path_dynamic` / `static_turning_circle` / `static_clearance` / `anthropometric_percentile` / `field_observation` / `participatory_spatial` / `stated_unmeasured`), `device_class` CHECK (`manual_self_propelled` / `manual_attendant` / `power_chair` / `scooter` / `bariatric` / `walker_rollator` / `mixed` / `not_device_scoped`), `root_population_note` (the root's sample description — the fit-population test), `file_anchor` (path+section into the md/yaml/json corpus — the tri-format provenance link). Derived view `v_value_independence`: per (parameter, population), COUNT(DISTINCT root_id) excluding `committee_assertion` — **the computable best-practice-vs-consensus discriminator.**

**Adjudication doctrine encoded (the six tests, applied per extracted value):** Root (did this document generate evidence or only transmit it?), Independence (do agreeing sources share ancestry?), Paradigm-commensurability (values corroborate/contradict only within a measurement paradigm), Fit-population (is the root's sample representative of this population's current device fleet/demography?), Currency (is the root superseded?), Interest (who convened the consensus?). Cross-class and cross-paradigm applications are recorded as *derivations*, never as identities.

### H2 — Derivation-path dimension on determinations

Extend the migration-026 `evidence_cell_state` design (0 rows; cheap now, expensive after backfill) with `functional_basis` (mechanism type per FDA §3 taxonomy + ICF code(s) + supporting refs incl. FDR file anchors) and `derivation_paths` CHECK (`dual` / `population_only` / `function_only`). Rule, in the exact shape of the existing `single_axis` rule: single-path assertion is permitted **only with a named-path rationale**; dual-path convergence is recorded as itself evidence (doctrine #3 extended). **Cultural/dignity-claim protection (hard rule, per Q9 and the CRPD critique):** claims whose normative force is community-rooted (e.g. signing space as linguistic-cultural need) remain fully assertable as `population_only` — the handshake makes the derivation *explicit*; it never requires a mechanism where the claim is cultural, and it never lets a functional derivation flatten a community claim (no bottom-up override of Co-1).

### H3 — Materialize the population↔function mapping as data

One table (`population_icf_links`): the FDA §2 mapping + armature Q12 mapping-confidence classifications, promoted from skill prose to provenance-bearing rows (population_code × ICF d-code × mechanism × confidence × provenance). This is simultaneously the forward map (population → characteristic function) and the substrate for the Q4 reverse-mapping algorithm (axis/ICF profile → relevant populations) — closing, not reopening, the open A3/A7 joint decision. Immediate payoff: reverse-mapping from d310/d465 would have flagged DEAF's absence from E-08's population links mechanically.

### H4 — Wire the existing advisory checks into the determination engine as gates

FDA Q4 verdicts UNLINKED/MISLINKED/UNDER-CONSERVATIVE and FDR delta-classification CONTRADICTS force the affected cell to `provisional` (never `stated`) until resolved — converting today's advisory gap entries into the agreement mechanism. FDR runs remain commissioned work; the gate binds only where a check has actually run (no silent pretence of coverage).

### H5 — The harvest pipeline (scattered → usable, tri-format)

Codify the flow the owner asked for: **md is where evidence is authored** (BPCs, FDR — narrative, citable, human-reviewable); **the DB is where evidence is queryable** (extraction rows, genealogy, determinations, gaps); **yaml/json are interchange only** — loaded or retired. Concretely: (a) `data/jurisdictional_values/*.yaml` → already loaded by the 026 backfill (A2/C1); add an `instrument_status` dimension per unification-DR item 15; (b) the six `references/tier*/co*-verified-sources.json` registries (551 entries) → one-time reconciliation audit against `evidence_sources` (disagreements are findings), then redirect-stub retirement per the connection-register precedent; (c) FDR markdown → the promised-but-never-executed merge lands as *extraction rows + `functional_basis` links*, not prose duplication into BPCs; (d) gap-filling is engine-driven: computing all cells turns every hole into a queryable `pending` row with category and falsification condition — the gap register becomes a work queue, not a memory. Sequence: A2/C1 → B6+H1 → extraction pass over the md corpus → engine backfill → registry retirement.

### H6 — Naming hygiene

"FDR" collides (Functional Deficit Research in `references/fdr/` vs Failure-Demand-Recovery in `schemas/fdr_specialist.py`). Rename one on ratification (recommend the schema module → `failure_demand_recovery.py`; the reference corpus keeps the acronym it prints in 38 filenames).
