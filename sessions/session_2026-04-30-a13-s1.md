**scripts/doctrine_recheck.py** (NEW, 14K / 372 lines): mechanical passes 2.2–2.5. Pass 2.2 walks `governance/*.md` for `**Status:** CANONICAL` headers, `references/project-standards.md` for `RULE:` blocks containing CANONICAL, `data/decisions/decision_register.yaml` for `status: ACTIVE` records, `references/bpc/**/*.md` for BPC count. Pass 2.4 loads prior snapshot, computes governance-doc and decision-ID set differences, ERROR-fires on silent governance-doc removal. Pass 2.5 lenient pre-A12-S2: empty register suppresses RULE-coverage check with INFO finding. Findings use placeholder finding_id RC-000-NNN until promoted to a real recheck_id at finalisation. CLI: `--snapshot-only` writes snapshot only; `--cross-ref` runs 2.2–2.3 only; `--baseline-against PATH` overrides default-most-recent prior snapshot; `--report` exits 0 always.

**scripts/contamination_sampler.py** (NEW, 7.5K / 215 lines): deterministic stratified sampling. Discovers topic groups under `references/bpc/`. Allocates samples across groups with floor=1 per non-empty group, then proportional excess by group population, with rounding top-up to largest groups. Selects first-alphabetical filename within each group's stratum (reproducible). Writes sampling manifest with `disposition: PENDING` per BPC for human classification. Warns on N<15 (Stage 0.4 methodology minimum). CLI: `--n`, `--recheck-id`, `--output`, `--corpus-root`. Exit 0 / 2.

### Bugs caught and fixed in-session

1. **Sampler `global` syntax error.** Original code had `global BPC_ROOT` after a use of the name — Python rejects (SyntaxError "name used prior to global declaration"). Fix: refactored `discover_topic_groups` to take `corpus_root` as a parameter; dropped the global write entirely. The sampler now passes `args.corpus_root or BPC_ROOT` through cleanly.
