## Completed

### Within-Candidate-B sub-substrate: SQLite operative

**SQLite** is the operative sub-substrate for Session 4's evaluation. PostgreSQL and DuckDB are dismissed within Candidate B per the rationale in the deliverable §1.2:

- SQLite: lowest-risk for solo-author posture (D-03); single-file format; CLI ubiquitous; SQLite Archive Format is a Library of Congress preservation recommendation; format longevity is the strongest of the three; corpus volume well within SQLite's operational sweet spot.
- PostgreSQL: server process adds operational complexity that pre-launch solo-author does not need.
- DuckDB: file format longevity is materially weaker than SQLite (younger project, single steward, format breaks have occurred); analytical-query strength does not compensate at the project's data volume.

Captured as **D-0129 D-METH/DG-REVIEW** (PENDING). Sessions 7–9 may revisit if Candidate B is selected and a different sub-substrate's properties later dominate.
