Corridor clear width How wide must a corridor be — and which function is setting the number?

Plan dimension governing whether people can travel, pass, turn, and hold a signed conversation along a route. The right number depends on which of those functions the corridor must serve — not on how many codes agree. ⚠ opus · editorial framing

◐ provisional value · evidentiary hold Every value on this page is PROVISIONAL. The underlying evidence corpus is in a deliberate RETRACTED-PRE-REHAB hold (68/82 best-practice slices), and the determination engine that will render adjudicated values from a genealogy substrate is not yet built. This page is a hand-authored re-derivation from the project spine (value-genealogy worked example, DR-ACCEPTED 2026-07-13; tier system §3, §8) — it replaces an earlier version of this file that carried a fabricated citation anchor. See § 10 · Evidence base for the integrity record.

§ 1 · Diagram & detail

One corridor, four functions, four widths

plan + section · pass and fail at parity

A corridor is a stack of requirements, not one. The plan below shows the four functional widths nested; only the outermost (2440 mm) carries a traceable measurement root (§ 5). The section shows how clear width is measured — against the device's swept envelope, floor to 2050 mm AFF, projections deducted.

§ 2 · Summary

Which function is setting the width

~40 sec · plainlang_v4
◇ plain-language summary

A corridor is not one requirement — it is a stack of them. At 915 mm one manual wheelchair fits (the US floor, set for manual chairs). At about 1200 mm one powered wheelchair fits with clearance (the UK / ISO minimum). At about 1800 mm two devices can pass (nearly every code agrees — but no one can point to the study behind it). And at 2440 mm two people can walk side by side and hold a signed conversation, and a powered chair has room for the path it actually sweeps when it turns. Only that last number has traceable measurement behind it. So the Guidebook's best-practice value on a primary route is ≥ 2440 mm — with the honest caveat that the figure comes from one Deaf-community source and one signed language (ASL), and has not been independently replicated.

The doctrine that decides this page: best practice ≠ convergence. That BS 8300, DIN 18040-1, AS 1428.1 and ISO 21542 all specify 1800 mm is a convergence claim across Tier 4–6 codes. Convergence tells you jurisdictions agree; it is not best-practice evidence — they may all be reading from one shared pre-evidence floor. Precedent counts documents; evidence counts roots. 1800 mm has the most documents and zero traceable roots; 2440 mm has two independent roots (tier-system §3).

§ 3 · Populations affected

Most populations' corridor strand is not a width

fan-out · variable-conflation test applied

The genealogy method's key result for this item: ask 22 population codes "what is your corridor requirement" and most answer with a different physical variable that happens to share the same architectural element. Flattening them into one fought-over millimetre figure is the error the method exists to prevent. Primary (width-anchoring) means the population's evidence actually sets a width; the rest inherit, or key a non-width variable. ◆ worked example §5 per-population fan-out

DEAF holds the anchor evidence for this item — and was dropped from it. The live DB item_population_links for E-08 list MOB/VIS/DEM/SCI but omit DEAF, even though DEAF supplies the strongest (2440 mm) width evidence. Top-down, hand-maintained links dropped exactly the population whose evidence is strongest. DEAF is restored as a primary anchor here.

◆ width-anchoring variable

Two signers walking in conversation with peripheral scan space: ≥ 2440 mm primary / 1830 mm secondary / 3050 mm exterior; interlocutor distance 900–3050 mm; glazed junctions; ≥ 300 lux diffuse, no backlighting; matte mid-tone walls for skin-tone legibility. ASL-derived, single-institution, LOW dimension confidence.

● primary · anchorMOB / SCI · Wheeled mobilitypassing + swept envelope + injury dosewidth›
◆ width-anchoring variable

Two-device passing (1800 floor) and the powered-chair swept envelope (up to 2400 mm entire-sample, Steinfeld 2006). SCI adds propulsion-effort / injury-dose. This is the width strand alongside DEAF; the numbers converge upward on 2400+.

◇ topology, not widthDEM · Dementialoop topology & ≤1 visible choice pointtopology›
◆ real variable

The DEM corridor variable is loop topology and ≤ 1 visible choice point (Marquardt, T3 root) — not width. The once-asserted "narrow corridors calm" reading was retired as a vestigial framing assumption; the Danish loop-agitation claim is a broken genealogy (source unverifiable). DEM inherits a width only incidentally.

◇ load, not widthAUT · SENS · NDV · ADHD · Neurodivergentsensory load is independent of widthindependent›
◆ variable-independence finding

The CORRIDOR-W conflict was retired 2026-03-30: "width and sensory load are independent variables." No study shows wide corridors harm; the real variables are RT60 ≤ 0.6 s, transition zones, visual noise. A ≥1500 mm proximity-buffer figure is Co-2-only and unpromoted. Listing NDV as a "primary author" of the width value (as the prior page did) is incorrect. The genuine conflict is with open sightlines, not width (§ 8).

◇ route metric, not cross-sectionOFS · PAIN · CFS · POTS · MSrest interval & distance budgetinginterval›
◆ real variable

The strand is rest-interval and distance budgeting (pain-ofs resolved to 25–30 m / median 27.5 m; the cross-BPC tension with the OFS BPC's ≤ 50 m is unadjudicated), interacting with width only via the alcove rule: rest seating recessed ≥ 200 mm off the clear width so it never subtracts from passing space (Co-2 root). Width-neutral by geometry.

◆ variable

Sighted-guide pairs walk shoulder-to-shoulder, so VIS inherits a linked-two-person width. VIS's own primary axis is detection and edges, not cross-section; a shoreline / edge-following-vs-excess-width hypothesis is logged as an external-mining target, not asserted.

§ 4 · Capacities addressed

The functional capacities this width serves

ICF activity domains · design addresses deficits

The provision is framed against what a person is trying to do, not a diagnosis. Each capacity below demands a different width; the corridor's target is the widest capacity it must support. Corpus caveat: the model does not yet carry a queryable ICF activity axis (one incidental d-code in the whole DB) — the mappings below are re-derived from the handshake in the worked example §6, not read from a stored field.

d465Moving around using equipmentwheeled mobility & the swept turn→ swept envelope›
◆ what the width buys

Room for the path a powered chair actually sweeps (footplate flare 250–500 mm+; drive-system-dependent envelope shape), not an idealised turning circle. Below the swept envelope the user is forced into the high-effort spin turn — an injury dose (§ 8). Keyed d465 + device class in the bottom-up derivation (FDR-BAB-04).

d450 / d455Walking & moving aroundtwo-way passing · companion-on-arm→ 1800 passing›
◆ what the width buys

Two devices, or a walker plus companion-on-arm, moving in opposite directions without one backing up. This is the capacity the 1800 mm code floor addresses — necessary but not the design target where signed conversation is in scope.

d310 / d330Communicating — receiving & producing signed messagesconversation while moving→ 2440 signing pair›
◆ what the width buys

The capacity nearly every code ignores: two people signing side by side with peripheral scan space, sightlines unobstructed across ≥ 3 m depth. This is the capacity that anchors best practice, keyed d310/d330 + DEAF in the bottom-up derivation (FDR-DEAF-01, flagged NOVEL — "absent from BPC"). Its normative force is linguistic-cultural (community-rooted), and the handshake records that without demanding a biomechanical mechanism.

d410–d429Changing & maintaining body position180° turn, transfer turn→ manoeuvre type›
◆ what the width buys

The turn is not one event: 90° corridor entry (median daily turn 39°, IQR 24–67°, Sonenblum 2012) ≠ 180° in an enclosed bay (2400 mm entire-sample) ≠ open bay (1925 mm) ≠ transfer turn. Values are commensurable only within a manoeuvre type + drive system — see § 5.

§ 5 · Best-practice targets

Performance targets & dimensions, by evidence strength

weighted-strength model · tier-system §8

Rows are ordered by how strong the evidence behind the number is (tier-system §8: ● full anchors outright · ◐ partial = standards basis, not primary evidence · ○ weak = floor/convergence, not academically adjudicated). The strongest number is not the largest code minimum — it is the one with a traceable measurement root. ◆ re-derived from value-genealogy worked example §1

● full · anchored Primary routes — best practicesigning-pair envelope + powered-chair swept path ≥ 2440 mm · 8 ft ›
◆ value

Primary routes ≥ 2440 mm. Secondary routes ≥ 1830 mm. Exterior routes where signing groups are anticipated ≥ 3050 mm. Signing interlocutor distance 900–3050 mm by group size. Sightline depth ≥ 3 m unobstructed.

◆ evidence · ● full

Two independent roots: DeafSpace/ASL signing-pair proxemics (Gallaudet 2010, Co-1) + measured swept-path envelopes reaching 2400 mm entire-sample in an enclosed bay (Steinfeld 2006, T3). Corroborated by Vaughn 2018 and Cloete & Rout 2025.

▲ honesty markers

Dimension confidence LOW — single institution, single signed language (ASL). No independent replication of the 2440 mm figure exists; treat as an ASL-derived floor (likely conservative for BSL/DGS, possibly insufficient for multi-person ASL groups), not a validated universal. The Steinfeld 2006 swept-path root is UNVERIFIED-1. Directional principles MODERATE confidence.

○ weak · convergence Two devices passingregulatory floor — code convergence, no traceable root ≥ 1800 mm ›
○ evidence · ○ weak

BS 8300-2, DIN 18040-1, AS 1428.1 and ISO 21542 converge on 1800 mm. Per tier-system §3 this is a convergence claim across Tier 4–6 sources — informative about regulatory agreement, not best-practice evidence. No upstream derivation exists in the corpus. Adopt as a floor where 2440 mm cannot be achieved; never relabel "preferred."

◆ device-class ladder A value rooted in one device class is not evidence about anothermeasured swept envelopes, not turning circles by class ›
◆ the ladder

Manual self-propelled — 1500 mm floor / 1700 mm best practice (+ footplate sweep). Powered — 1700–1800 mm by drive type (ICC A117.1-2017 concedes 1702 mm powered; chair width 700–800 mm vs ADA's 915 mm, "set for manual passage only"). Scooter — 1800 mm+, multi-point Ackermann envelope unmodelled. Bariatric — 1900 mm new-build. Children's powered — MEXT ≥ 2400 mm. Attendant-propelled — total corpus silence.

● named by the codes themselves

PT DL 163/2006 and KR 편의증진법 state their 1500 mm figures are manual-wheelchair-based; SG BCA Code 2025 explicitly recognises prior code insufficient for contemporary device footprints. Cross-class application is a derivation (recorded, down-weighted), never an identity. Base anthropometry stale (GAP-295).

§ 6 · Relevant products

Components & products the provision implies

component classes · not a product database

The corpus holds no product-SKU register, so these are component classes implied by the provision, each tied to a requirement established above — not brand recommendations. Two rules govern all of them: continuous projections below 2050 mm AFF deduct from the clear width, and anything that can creep into the clear zone post-occupancy must be wall-recessed. product-class list is editorial, derived from the schedule spec + DeafSpace finish spec; no SKU-level source exists.

◆ handrailsContinuous handrailsprojection deducts from clear width−50 mm typ.›
◆ spec impact

Continuous rails count against the clear width even though intermittent in plan (a wheelchair cannot pass through them): 50 mm typical stainless, 60–75 mm brass/timber. Wall-recessed grab rails do not count. Size the wall-to-wall so the clear dimension survives the deduction.

◆ glazingGlazed junction corners / vision panelsDeafSpace visual advance warningat junctions›
◆ spec impact

Glazed corners replace opaque corners at all circulation junctions so an approaching person is seen in advance — the sightline half of the 2440 mm provision (see § 1 detail, and D-04 landmarks). Co-1 root: DeafSpace / DeafScape.

◆ finishes + lightingMatte medium-contrast finishes · ≥300 lux diffuseskin-tone legibility for signing / lip-reading≥300 lux›
◆ spec impact

Matte, medium-contrast neutral wall/floor finishes (muted greens/blues) against which skin tone is legible; diffuse, shadow-free lighting ≥ 300 lux at face height in communication zones; backlighting eliminated. Cross-ref B-08 floor finish, A-02 overhead. Confidence MODERATE (converging directional evidence).

◆ recessed FF&EWall-recessed fixturesextinguisher cabinets · sanitiser stations · call points0 mm proj.›

Fire extinguisher cabinets, hand-sanitiser stands, emergency call stations and signage below 2050 mm AFF must be wall-recessed so they do not eat the clear width post-occupancy (a common failure mode — § 8). Specify at FF&E review; mark the clear zone in floor-plate hatching.

◆ spec impact

The OFS/PAIN rest-interval strand (§ 3) meets E-08 only via the alcove rule: rest seating recessed ≥ 200 mm off the clear width so it never subtracts from passing/signing space (Co-2 root). See E-10.

◆ doors + furniturePowered doors & movable furnituredoor opening ≥ corridor; armrest-free chairs≥900 mm clear›
◆ spec impact

Door clear opening not narrower than the corridor (900 mm best practice; 850 mm min for power WC, CSA B651) — see E-11. In DEAF communication zones, furniture movable and without fixed armrests so users can form conversation circles (DeafSpace).

§ 7 · Considerations & questions

What to ask before you fix the number

design-brief interrogation

The width follows from the answers to these. The single most consequential is the first — because the building cannot infer which functions it must serve, the brief must state them.

  • Function scope Which functions must this route serve — single-device travel (1200), two-way passing (1800), or signed conversation-in-motion (2440)? If any Deaf/signing use is plausible, the target is 2440, not the passing floor.
  • Silent-deletion check Is the population scope "MOB accessibility" only? If so, does stopping at 1800 mm silently delete DEAF signing space (CON-0122)? Confirm DEAF is in scope explicitly — the live data dropped it (§ 3).
  • Device classes Manual, powered, scooter, bariatric, children's powered, or attendant-propelled? The ladder differs by class (§ 5); scooter and attendant-propelled envelopes are corpus-unmodelled — treat as open risk, not solved.
  • New-build vs retrofit Is 2440 mm achievable, or is a masonry constraint forcing the floor? If forced below best practice, document as a DAR design compromise, not as compliance.
  • Mixed-population enclosure conflict Does the facility also serve NDV/SENS or DEM users who need enclosure? Open DeafSpace sightlines conflict with compartmentalisation — resolve by Tier-1 spatial zoning, never a Universal-Mode split-the-difference width (GAP-CONF-01, § 8).
  • Measurement paradigm Is the clear width measured against the device's swept envelope or an idealised turning circle? Use swept envelope + manoeuvre type; a turning-circle number cannot adjudicate a swept-path requirement.
  • Anthropometric currency Is the governing dimension from current device-fleet data, or a 1970s/2010 base now stale against heavier chairs and scooters (GAP-295)?
  • FF&E & seating placement Where will fixtures and rest seating sit relative to the clear zone, and are they recessed so they cannot creep in post-occupancy (§ 6, § 8)?
  • Cross-language validity If the Deaf user community signs a language other than ASL (BSL, DGS, LSF, LIS…), note that 2440 mm is an ASL-derived floor with no cross-language measurement — record the assumption rather than treating it as settled.
§ 8 · Failure cases

How this provision fails in practice

7 modes · design + evidential

Failure modes, physical and evidential. Each is a real, corpus-grounded way the provision goes wrong — none rely on the fabricated metabolic figures the previous version of this page invented.

✗ FAIL-1Single-file pinch"one chair fits" mistaken for "the corridor works"at 1200›
▲ mode

A corridor at the 1200 mm single-device minimum admits one powered chair — and nothing else. Two devices cannot pass (VNG 2016: 1200 mm insufficient to pass or turn); signed conversation collapses to single file. Meeting the minimum is not meeting the requirement.

✗ FAIL-2Forced spin-turn — injury dosewidth below the swept envelopeRSI›
▲ mode

Below the swept envelope the user is forced into the inferior spin turn — inner-hand braking ~15× a straight push, hundreds of times a day, against a manual-user upper-limb injury prevalence of 55–72%. A cramped corridor is a repetitive-strain-injury dose, delivered for the user's whole wheelchair-using lifetime.

✗ FAIL-3Silent deletion of DEAF spaceMOB-only spec stops at the passing floorCON-0122›
▲ mode

A designer specifying "MOB accessibility" and stopping at 1800 mm deletes the 2440 mm signing envelope without ever seeing it — and the dropped item_population_links row for DEAF (§ 3) makes the deletion invisible in the data too. The subsumption 2440 ⊃ 1800 is the guard: meet the larger.

✗ FAIL-4Open sightlines vs enclosureDeafSpace ⟂ NDV/DEM in a mixed facilityGAP-CONF-01›
▲ mode

Delivering 2440 mm via open sightlines produces visual complexity and lack of enclosure contraindicated for NDV/SENS (ASPECTSS crit. 4) and DEM (bounded sections). Trying to resolve it with a compromise width at Universal Mode fails both — the resolution is Tier-1 spatial zoning (Deaf communication zones vs retreat zones), evidence LOW for dimensions / MODERATE for principles.

✗ FAIL-5FF&E creep & door pinchthe clear zone lost post-occupancy−width›
▲ mode

A fire extinguisher, sanitiser stand or planter added after handover projects into the pinch point; or a door clear opening narrower than the corridor reintroduces the pinch the width was meant to remove (E-11). Both defeat a compliant plan. Mitigation: recessed FF&E (§ 6), hatched clear-zone at review, post-occupancy walk-through.

✗ FAIL-6The "narrow calms" design mythapplying a retired assumption to DEMretired›
▲ mode

Deliberately narrowing corridors "to calm" dementia or neurodivergent users applies a claim the corpus retired (a broken genealogy; "width and sensory load are independent variables," 2026-03-30). The real DEM variable is loop topology; the real NDV variables are acoustics and visual noise. Narrowing helps neither and harms mobility.

✗ FAIL-7Convergence sold as evidence"1800 is best practice because every code agrees"method›
▲ mode

The named failure mode of tier-system §3: labelling the 1800 mm code convergence "preferred" or "best practice." It is the regulatory floor with zero traceable roots. This is the exact error the previous version of this page committed by calling ≥1200/1800 "the Guidebook value."
