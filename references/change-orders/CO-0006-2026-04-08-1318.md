## Scope

This Change Order authorises structural modifications to the infrastructure of the Accessible Built Environments Guidebook. These changes affect the BPC schema, connection register architecture, search log schema, new reference files, a new skill, and updates to six existing skills.

**No existing BPC synthesis content, item specifications, or Part 1–12 prose is modified by this CO.** Schema changes are applied on first-touch migration for existing files (not mass-migrated). New deliverables are additive.

**Principle:** Fix the pipes before pumping more water. Integration throughput — not additional research volume — is the critical path constraint this CO addresses.

## 1. BPC Key Sources Schema Upgrade

**Current state:** BPC entries record source identifiers only (e.g., `ADA-2010-S404`, `Brandt-2016-PMID28656108`) in a flat list under `### Key sources`. Full bibliographic metadata is not recorded at research time and is being permanently lost. GAP-CR-01 (empty bibliography) is a downstream symptom.

**Authorised change:** Replace the flat Key sources list with a structured metadata table in all BPC entries.

### New schema

```markdown
### Key sources

| REF-ID | Short-key | Authors | Year | Title | Journal/Publisher | DOI/URL | Tier | Lang | Jurisdictions |
|---|---|---|---|---|---|---|---|---|---|
| 01 | ADA-2010-S404 | US Department of Justice | 2010 | 2010 ADA Standards for Accessible Design | Federal Register | ada.gov/law-and-regs/... | Tier 5 | EN | US |
```

**Field definitions:**
- `REF-ID`: Sequential integer (01, 02, …) — stable within BPC entry; referenced by ISW `#### Sources cited` tables
- `Short-key`: Current identifier format — retained for backward compatibility
- `Authors`: Surname, Initial; "Organisation Name" for institutional sources
- `Year`: Publication year; `n.d.` if unknown
- `Title`: Full title; truncate at 120 chars with `…` if needed
- `Journal/Publisher`: Journal name, publisher, or issuing body
- `DOI/URL`: DOI preferred; URL if no DOI; `[GREY]` if no persistent identifier
- `Tier`: Evidence tier per project hierarchy
- `Lang`: ISO 639-1 source language code
- `Jurisdictions`: Comma-separated jurisdiction codes

**Migration policy:**
- Do NOT mass-migrate all BPC files in a single pass
- Apply when each BPC is next touched (Opus synthesis, ISW run, or dedicated migration session)
- Tier 5 (standards/regulatory): deterministic metadata — Haiku batch extraction viable
- Tier 1–3 (academic): PubMed/CrossRef API lookup by PMID/DOI via citation-verifier `ENRICH` mode (§4 below)
- Grey literature, Tier 4, Tier 6: manual metadata entry at research time

**Skills affected:** research-log-manager, multilingual-research (§5 below)

## 2. Connection Register Restructuring

**Current state:** `references/connection-register-active.md` is a single 197K-character, 3,480-line file. No model context window can load it alongside a BPC entry and item specification simultaneously. This makes connection application structurally bottlenecked by context capacity (75 PENDING + 58 SPECULATIVE connections affected).

**Authorised change:** Split the connection register into a topic-directory architecture mirroring BPC/SL structure.

### New directory structure

```
references/connections/
  _index.md
  bathrooms-and-wet-areas/connections.md
  communication-and-alerts/connections.md
  controls-and-hardware/connections.md
  economics/connections.md
  entrances-and-circulation/connections.md
  frameworks-and-methodology/connections.md
  health-and-symptom-management/connections.md
  kitchens-and-workspaces/connections.md
  population-general/connections.md
  room-types/connections.md
  seating-and-rest/connections.md
  sensory-environment/connections.md
  wayfinding-and-signage/connections.md
  cross-cutting/connections.md
```

### Routing rules

1. A connection is filed under the topic directory of its **primary target item** (the item that needs to change).
2. If a connection links items in two directories with no single primary target: file under the directory containing the item with the **higher item-code letter** (later in alphabet = later in specification sequence = more likely to accommodate).
3. If a connection spans 3+ directories, or routing remains ambiguous after rules 1–2: file under `cross-cutting/`.

### `_index.md` schema

```markdown
| CON-ID | Status | Primary target | Filed in | Confidence | Opus-reviewed | Session applied |
|---|---|---|---|---|---|---|
| CON-0001 | CONSUMED | D-01/D-02 | wayfinding-and-signage | HIGH | Yes | session_2026-03-26 |
```

### Per-entry schema (within topic files)

```markdown
### CON-{NNNN}

**Status:** PENDING | CONSUMED | DEFERRED | SPECULATIVE | CLOSED
**Confidence:** HIGH | MODERATE | SPECULATIVE
**Opus-reviewed:** true | false
**Source BPC slug(s):** {slug1}, {slug2}
**Target item(s):** {item-code(s)}
**Target population(s):** {pop-codes}
**Evidence tier:** {tier}
**Filed:** {YYYY-MM-DD}
**Applied:** {session ref or —}

**Connection:** {1–3 sentence description}

**Evidence basis:** {brief citation from BPC}

**Action required:** {what ISW needs to do}

**Disposition notes:** {resolution notes, deferred reasons}
```

**Migration process:**
1. Parse `connection-register-active.md` into structured entries (Sonnet, 1 session)
2. Route each entry per rules above
3. Build `_index.md` from parsed data
4. Archive `connection-register-active.md` and `connection-register-archive.md` to `_archived/`
5. Update connection-scout_SKILL.md to write to new structure
6. Update workplan-orchestrator_SKILL.md session-start protocol to load `_index.md` + relevant topic file(s) only

**Skills affected:** connection-scout, workplan-orchestrator (§5 below)

## 3. Search Log Disposition Field Addition

**Current state:** Search log entries record search actions but do not capture the downstream disposition of retrieved sources (whether results were integrated into BPC, deferred, or discarded).

**Authorised change:** Add a `disposition` field to search-log entry schema.

**Field values:**
- `INTEGRATED`: Sources incorporated into BPC Key sources table
- `DEFERRED`: Sources retrieved but not yet integrated; reason recorded
- `DISCARDED`: Sources retrieved but excluded; reason recorded
- `PENDING`: Search complete; disposition not yet determined

**Application:** New searches from this CO forward. Existing entries: flag `[PRE-CO-0006 — no disposition]` on first review. No retroactive mass-update.

**Skills affected:** research-log-manager, multilingual-research (§5 below)

## 9. Correctness Amendments to Workplan

The following amendments apply to the source workplan before execution:

**C1 — PMID/DOI coverage estimate:** Workplan states ~40% coverage. This is unverified. Before Phase 0A-2 execution, scan all BPC Key sources for PMID/DOI pattern matches and report actual percentage. If <20%, manual metadata entry is the primary strategy and 0A-2 scope narrows accordingly.

**C2 — Connection routing (already incorporated):** Routing rule for ambiguous two-directory connections uses higher item-code letter as tiebreaker. Incorporated into §2 above.

**C3 — Conditional specification values:** Specification database schema (Phase 2A) must include a `conditions` array for parameters with population-dependent values (e.g., 1500mm single wheelchair user; 1800mm two users passing). Top-level `value_min`/`value_max` represent full range across all conditions.

**C4 — Term co-occurrence quality (Phase 3A Option C):** Exclude standard and regulation identifiers from term extraction. Focus on: physiological mechanisms, anatomical terms, functional deficit descriptions, spatial relationship terms, material/performance terms. Standard numbers are cross-referencing artifacts, not substantive similarity signals.

**C5 — Classification system versions (Phase 4B):** Use Uniclass 2015 (UK), CSI MasterFormat 2018 (NA), NATSPEC 2024 (AU). Cross-reference must state edition and include a last-verified date.

**C6 — Bedroom slug scope (Phase 5A):** Ceiling hoist provision already covered in `upper-limb-impairment-built-environment` BPC. Bedroom slug must cross-reference rather than duplicate. Bedroom slug scope: room-level spatial organisation, furniture layout, lighting zones, proximity relationships, DAR provisions, sensory environment — not individual fixture specifications.

**C7 — Session estimate revised:** 35–54 sessions (up from 28–43). Phase 0A migration adds ~0.5 sessions per 10 BPC files touched during v10-5 content production (~4.5 sessions total marginal overhead, mostly absorbed into content sessions).
