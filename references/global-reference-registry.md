## Cross-Reference Notes

**BPC local REF-ID → Global REF-ID mapping:** The `local_ref_ids` column in each registry entry lists the original BPC-local IDs (e.g., `room-acoustic-performance:RAP-12`) that map to this global REF-ID. Phase B parsers build a `local → global` lookup from this column.

**Updating the registry:** New references append with the next sequential REF-ID. Never re-use retired IDs. If a reference is removed from all BPCs, mark the registry entry `status: RETIRED` — do not delete.

**DOI resolution:** Section 3 (GREY) items require DOI resolution before publication. See `gap_register.md` GAP-DOI-01 for status. When a DOI is resolved, move the entry from Section 3 to Section 1 and update `metadata_quality`.

**Authority:** This registry is the single source of truth. If a BPC Key sources table and this registry conflict, the registry governs.
