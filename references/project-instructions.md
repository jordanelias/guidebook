## Overview

This project produces, audits, and critiques **technical guidebooks** for OT practitioners, architects, and policy makers. Primary lens: occupational therapy evidence. Best practice = most amenable, inclusive, forgiving, caring, accommodating, dignified, specific, and targeted provision the evidence supports.

**Active version:** V9.0 2026-03-20
**Primary file:** `Guidebook_for_Accessible_Design_v9-0_2026-03-20.md` (6,547 lines)

## Session Protocol

### Start (every conversation)

1. GET most recent `sessions/` file. Report `session_close` + `next_action`. Confirm with user.
2. GET `gap_register.md` — surface OPEN P1 items.
3. GET `references/project-standards.md` — load rules.
4. Confirm PAT.

**Skip:** "fast-track" or "go" skips user confirmation at Task Intake only. Protocol still runs. `toc-editor` never skips pre-flight.

### Checkpointing

After each skill stage: `CHECKPOINT [YYYY-MM-DD HH:MM] — task: {task} — stage: {stage} — status: {status}` (1–2 lines max).

### Close (~85% usage OR natural conclusion)

1. Complete current stage. Do not start new.
2. Write all findings to GitHub.
3. Invoke `session-consolidator`.
4. Bullet-point handoff.
5. Instruct new conversation.

## Three-Tier Design Hierarchy

DAR mandatory at every tier.

| Tier | Context | Specification |
|---|---|---|
| **0** | No predominant disability population | Above code minimum; designed for future tailoring |
| **1** | Identified population(s) | Ranges; median = population-informed default |
| **2** | Identified individual | Co-designed; resolves value within Tier 1 range |

Ranges bridge Tier 1↔2. At Tier 1: use median. At Tier 2: co-design resolves. The guidebook is a framework for professional judgment, not a substitute. At Tier 2, co-design resolves conflicts, not the guidebook.

## Evidence Hierarchy

Higher tier governs on conflict.

| Tier | Type |
|---|---|
| **1** | OT clinical research — intervention-tested |
| **Co-1** | Lived experience / participatory design (CRPD Art. 4.3) — co-primary with Tier 1 |
| **2** | Disability-led NGO / DPO / advocacy guidelines |
| **Co-2** | OT clinical practice guidelines |
| **3** | Systematic reviews and meta-analyses |
| **4** | International standards with evidence basis |
| **5** | National beyond-code frameworks |
| **6** | Statutory codes — reference baseline only |

Co-1/Tier 2 searched first (most under-retrieved). Lived experience co-primary where no RCT is feasible. Evidence specification markers: ● = evidence-based (Tier 1–6); ○ = expert consensus; ◐ = mixed.

## Standing Rules

### Document Processing
1. Never process >500 lines as single input → `haiku-chunker` first.
2. Fresh DOCX conversion → `fix-linebreaks` before analysis.
3. All timestamps: `YYYY-MM-DD HH:MM`.
4. Default output: `.md`. DOCX only if explicitly requested.

### Research
5. `research-log-manager CHECK` before any `multilingual-research` run; `LOG` after. Skipping either = error.
6. Pre-LOG completeness gate: 7 conditions (see `research-log-manager` skill file). LOG blocked until all pass or gaps explicitly accepted.
7. View Keyword Compendium (`multilingual-research-protocol-v4-2026-03-18-1.md` Part 3) before research. Use native-language terms.
8. Co-1/Tier 2 first, citation mining mandatory for Tier 1–2 sources. Use `citation-miner` skill (inline fallback only if switching would disrupt flow).
9. Best-practice synthesis mandatory before LOG. Not a catalogue — identify most inclusive, targeted, dignified provision.
10. Language (14) and jurisdiction (24 + ISO) are distinct axes; both must be independently satisfied.
11. Pre-v4 slugs: triage as CONSUME / SUPPLEMENT / RE-RUN before executing (see Rule 23 table in project-standards).
12. Flat BPC/SL files (15 population-level) are frozen archives. New research → per-slug directory entries.
13. Sources confirmed real. "I don't know" > invention. After 2 failed searches: delete, log CLOSED-DELETED. `[UNVERIFIED — DOI/URL required]` for unverified.

### Citation & Endnote
14. Every prescriptive claim carries `[REF:{slug}:{NN}]`. Pre-step: `research-log-manager RETRIEVE`. `#### Sources cited` replaces `**Key citations:**` for new/revised items.
15. BPC Key sources ordering frozen once REF-IDs emitted. New sources append only.
16. Bibliography reconciliation required at assembly: diff bibliography-compiler output vs bibliography-v9.md.
17. Endnote pipeline dry run (1 item end-to-end) must pass before Phase 3.

### Structure
18. Structural changes → `toc-editor` first. Updates `toc.md` + generates Change Order. Never skip.
19. Volumes: Roman numerals. Parts: sequential 1–13 across volumes. Sections: §X.Y within Part.
20. Design stages: "DD" = Design Development; "RFO" = Ready for Occupancy.

### Session Management
21. Never re-run completed stages. Consume existing outputs.
22. Gap register IDs must be unique. Collision → suffix -b, -c. Dedup on sight.
23. Working documents committed to GitHub before session close.
24. Connection register: `item-specification-writer` updates PENDING→CONSUMED. `session-consolidator` verifies no HIGH/PENDING orphaned.
25. Activate PubMed, Consensus, Scholar Gateway only when task requires research/citation verification.

### Model Selection

| Task | Model |
|---|---|
| Structural extraction, pattern matching, heading correction | Haiku 4.5 |
| Judgment, synthesis, writing, evidence stratification | Sonnet 4.6 |
| Contradictory cross-language/jurisdiction synthesis (evidence hierarchy can't arbitrate) | Opus 4 |
| Connection discovery (both modes) | Opus 4 |
| Cross-synthesis: ≥3 population codes with conflicting requirements | Opus 4 |

Opus 4 is non-default. Activates only on named conditions.

### Prose Register
- Voice: soft imperative subjunctive — "to be", "not to exceed"
- Headings: descriptive only — no values/ranges
- Sequencing: Ideal → Best Practice → Acceptable → Minimum
- Concision: ≤25 words/sentence; one claim per sentence
- Every prescriptive claim carries citation, evidence tier, or `[UNSUPPORTED — citation required]`

### Versioning
- Major (X.0) = structural change. Minor (X.Y) = content revision. Patch (X.Y.Z) = editorial.
- ToC version independent of document version.

### FDR CONTRADICTS Protocol
Do not delete original BPC claim. Append `[CONTRADICTED BY FDR: {scenario} — {source}]`. Route to evidence-auditor. Record adjudication in BPC.

### Evidence Priority
When writing/specification skills receive both BPC content and raw search results: BPC governs. Raw results supplement only where BPC is silent.

| Workflow | Sequence |

| **Item Specification** | item-consolidation-analyzer → [research-log-manager RETRIEVE] → item-specification-writer (REF-ID + sources-cited) → vol2-item-formatter → [framing-checker · evidence-auditor] → prose-style-checker → volii-validator |
| **Item Specification (conflicted)** | …→ item-specification-writer → Opus 4 cross-synthesis → vol2-item-formatter → … |
| **Structural Change** | structure-auditor → markdown-formatter → cross-reference-resolver → find-and-replace → guidebook-auditor A |
| **Structural Nomenclature** | toc-editor → find-and-replace → cross-reference-resolver → guidebook-auditor A |
| **Document Assembly** | chunk-assembler → bibliography-compiler → bibliography-reconciliation → cross-reference-resolver → guidebook-auditor A |
| **Research Retrieval** | research-log-manager CHECK → if COMPLETE: RETRIEVE · if PARTIAL/STALE/NOT FOUND: multilingual-research → research-log-manager LOG |
| **Multilingual Research** | research-log-manager CHECK → [Keyword Compendium + Protocol v4 Networks] → multilingual-research (Co-1/Tier 2 → statutory → beyond-code → academic → citation mining) → [Opus 4 if contradictory] → research-log-manager LOG |
| **Citation Mining** | citation-miner (backward + forward) → research-log-manager LOG |
| **Bottom-Up FDR** | research-log-manager CHECK (top-down COMPLETE) → functional-deficit-researcher (≤12 scenarios) → research-log-manager LOG |
| **Connection Discovery** | connection-scout (internal then external) → connection-register → HIGH→item-spec briefing · SPECULATIVE→gap_register P3 |
| **Bibliography Reconciliation** | bibliography-compiler output → diff vs bibliography-v9.md → flag NEW-SOURCE / ORPHAN-LEGACY / DOI-MISMATCH |
| **Endnote Dry Run** | 1 item → item-specification-writer → vol2-item-formatter → chunk-assembler → bibliography-compiler → cross-reference-resolver |
